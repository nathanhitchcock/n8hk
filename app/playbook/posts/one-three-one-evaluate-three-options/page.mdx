Step 2 is where critical thinking shows up.

Most teams fail here by presenting one preferred option and two fake alternatives.

## Objective

Develop three genuinely different options and evaluate each against the same decision criteria.

## Option Template

For each option, document:

1. What this option is.
2. Expected upside.
3. Key risk.
4. Cost/time to execute.
5. Operational impact (team load, dependencies, customer effect).

Use the same template for all three options. If criteria change between options, comparison breaks.

## Build Real Option Diversity

Good option sets usually include:

- Low-risk / lower-upside path.
- Balanced path.
- Higher-risk / higher-upside path.

If all three options have similar risk and cost profiles, you likely only have one option with wording changes.

## Scoring Pass

A lightweight scoring pass keeps decisions honest.

Score each option 1-5 on:

- Impact
- Risk
- Time to value
- Reversibility
- Execution complexity

Scores are not the decision. They make assumptions visible.

## Manager Coaching Prompts

- "What makes Option B truly different from Option A?"
- "Which option fails fastest if assumptions are wrong?"
- "Where is each option brittle under stress?"

These prompts force better analysis without taking ownership away from the proposer.

<Callout title="Practical Guardrail">
  <p>
    Do not optimize for perfect certainty. The goal is a high-quality, explicit tradeoff set that supports a timely decision.
  </p>
</Callout>

## Done Criteria

This step is complete when:

- Three options are materially different.
- Tradeoffs are explicit and comparable.
- Risks and assumptions are visible.

Step 3 is choosing one option and owning the recommendation.
