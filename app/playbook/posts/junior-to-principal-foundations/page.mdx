The first leap is not title-driven. It is fundamentals-driven.

A junior engineer who wants principal-level growth needs repeatable technical habits before broad scope.

## Objective

Build durable engineering fundamentals that hold under pressure.

## Core Focus Areas

- Debugging discipline and hypothesis-driven troubleshooting.
- Code quality habits (readability, testability, maintainability).
- Reliability mindset (edge cases, failure modes, observability).
- Ownership mindset (follow-through beyond ticket completion).

## Weekly Practice Loop

1. Ship one meaningful improvement with tests.
2. Run one root-cause analysis of a bug or incident.
3. Document one technical decision and tradeoff.
4. Get feedback from a stronger engineer and apply it immediately.

## Signals of Progress

- Fewer regressions in shipped work.
- Better PR reasoning and clearer technical tradeoffs.
- Stronger problem decomposition before coding.

<Callout title="Guardrails">
  <ul>
    <li>Do not optimize for speed at the cost of understanding.</li>
    <li>Do not hide uncertainty during reviews.</li>
    <li>Do not treat testing as optional cleanup.</li>
  </ul>
  <p>Foundations are what make later scope survivable.</p>
</Callout>

## Done Criteria

This step is complete when the engineer consistently ships quality work with clear reasoning and minimal rework.
